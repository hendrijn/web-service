For whoever reviews my site:

GET /

ex. curl https://project2.jacquelyn-hendricks.me:8443/

Upon visiting https://project2.jacquelyn-hendricks.me:8443/ all orders within the database should appear. I have left an entry in the database so you can see how everything is organized.

GET /orders/:name

ex. curl https://project2.jacquelyn-hendricks.me:8443/orders/jackie

Gets an order from a specific person.

POST /orders

ex. curl --header 'Content-Type: application/json' --request POST --data '{"name": "Tony", "items": "Burger with Fries", "total": 6.99}' https://project2.jacquelyn-hendricks.me:8443/orders/

Adds an order to the database. Required fields are the customer name, the items they wanted, and their total (as a double).

PATCH /orders/:oldName

ex. curl --request PATCH --data '{"name": "Peter"}' https://project2.jacquelyn-hendricks.me:8443/orders/tony

Changes the customer's name for a given order. The name in the url is the current name for the order. The name in the JSON is the new name.

DELETE /orders/:name

ex. curl --request DELETE https://project2.jacquelyn-hendricks.me:8443/orders/kayla

Deletes an order from the database given the customer's name.

  1. My service manages a restaurant ordering system, not a to-do list.
  2. My service uses MySql to manage the database with one table to store all the orders.
  3. My service uses Express to interact with the database.
  4. My service is available at https://project2.jacquelyn-hendricks.me:8443/
  5. The endpoint URL's all have appropriate parameters that relate to the orders database.
  6. My service receives JSON data in the endpoints and sends JSON data as a response.
  7. My service can add orders to the system, get all orders, get an order by name, update the name of an order, delete all orders, and delete an order by name.
  8. I used ufw to block all ports but the one for SSH and the server. (80, 8443, 443 are allowed).
  9. I used a Nginx server to allow encrypted access to the service.
  10. My service supports cross-origin resource sharing by including middleware and the options wildcard endpoint.
  11. My service uses pm2 to keep the service running even when I am logged out of the droplet.
  12. My service is stored in the GitHub repository "web-service" and is shared with Dr. Johnson on Slack.
  13. The create_project2.sql and schema.sql scripts are in the repository. The credentials.json file is not included in the repository.
  14. My GitHub repository has a steady flow of commits.
  15. The service sends the html page for this report when the user visits https://project2.jacquelyn-hendricks.me:8443/report.html
